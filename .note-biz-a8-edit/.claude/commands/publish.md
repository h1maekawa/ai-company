# 公開記録

引数：`$ARGUMENTS`

これはnoteへ投稿するコマンドではありません。運用者がnoteへ手動でコピペして公開した後、その記録だけを行います。

## 手順

1. `CLAUDE.md`を読み、公開ファイル、価格、任意の公開URLが引数にあることを確認する。
2. 公開ファイルが`drafts/`内に存在し、価格が0以上の整数であることを確認する。
3. アフィリエイト候補やリンクがある場合、記事冒頭の広告表記と、`data/affiliate-links.csv`に登録された提携承認済みA8/Amazonリンクであることを確認する。
4. 次を実行する。

```bash
node scripts/publish.mjs <draftファイル名> <価格> [公開URL]
```

5. 実行結果を読み、公開直後チェックリストを表示する。

## 公開直後チェックリスト

- [ ] note上で有料ラインの位置を確認
- [ ] スマートフォン表示で見出し・コード・リンクを確認
- [ ] アフィリエイトリンクがある場合は記事冒頭の広告表記を確認
- [ ] X告知文を1つ投稿
- [ ] 関連マガジンへ追加
- [ ] `data/kpi.csv`の新規行を確認
- [ ] 24〜48時間後にPV・スキ・販売数を記録
