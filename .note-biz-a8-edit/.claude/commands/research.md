# note市場調査

対象テーマ：`$ARGUMENTS`

`CLAUDE.md`を最初に読み、ジャンル、読者、文体、遵守事項に従ってください。引数が空の場合は既定ジャンル「AI活用 × 営業・業務自動化」を使います。

## 手順

1. `research/`の既存レポートを確認し、同じ調査の重複を避ける。
2. Web検索で、対象テーマに関連する公開情報を調べる。note上の記事、検索需要を示す候補語、競合する解説記事、関連商品の価格感を複数ソースで確認する。
3. ログインや購入が必要な情報を見たと推測しない。「売れている」は販売数など公開された根拠がある場合だけ断定し、それ以外は反応数・検索露出・公開事例からの仮説と明記する。
4. 各事実にURL、ページ名、確認日を付ける。長い転載はせず要約する。
5. 今日の日付と安全な英数字slugを使い、`research/YYYY-MM-DD-<slug>.md`へ保存する。同名がある場合は上書きせず、内容を確認して連番を付ける。

## 必須構成

- 調査テーマと読者
- エグゼクティブサマリー
- 読者の課題と需要キーワード
- 有料note・競合コンテンツの観察
- 価格帯とタイトルパターン
- 一次情報として差別化できる前川の実務資産（未確認事項は要レビューと表示）
- A8またはAmazonの候補（自然に関連する場合のみ。提携承認前のURLは作らない）
- リスク・断定できない点
- 参照元一覧（URL・確認日）
- 次に書くべきテーマ3案

3案にはそれぞれ、タイトル案、想定価格、狙う読者、選定根拠を必ず含めてください。最後に保存先を表示してください。
