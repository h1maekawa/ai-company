# note記事執筆

テーマ：`$ARGUMENTS`

テーマが空なら、候補を捏造せず入力を求めてください。`CLAUDE.md`と`templates/article.md`、`templates/announce.md`を必ず読みます。

## 参照順序

1. `research/`から更新日時が最も新しいレポートを探す。
2. テーマに関連するレポートを優先し、該当するものがあれば必ず参照する。
3. `data/affiliate-links.csv`を読み、カテゴリに合い、`affiliate_url`が入力済みの候補だけを利用できる。空欄のURLや未登録URLを創作しない。
4. リポジトリ内に存在しない前川の経験、顧客事例、成果数値を創作しない。不足箇所は`[前川レビュー：実体験を追記]`と明示する。

## 出力要件

`drafts/YYYY-MM-DD-<slug>.md`へ、テンプレートの全セクションを保ったまま保存してください。

- タイトル案5つと採用タイトル
- 無料部分：800〜1200字を目安に、導入、課題共感、結論のチラ見せ
- `<!-- paid-content-start: note上ではこの位置から有料に設定 -->`を有料ラインに置く
- 有料部分：3000〜6000字を目安に、手順、コード、コピー可能なテンプレートを含める
- 推奨価格：初期300〜500円の範囲と具体的な根拠
- X告知文3パターン
- A8またはAmazonで自然に紹介できる承認済み候補がある場合だけ記載する
- アフィリエイトを使う場合は記事冒頭に`※本記事には広告・アフィリエイトリンクが含まれます。`を残す
- frontmatterの`slug`、日付、価格、`category`、`affiliate`、`affiliate_links`、`disclosure`を埋める
- カテゴリは`AI仕事術`、`AIキャリア`、`AI投資メモ`のいずれかにする

## 品質確認

- です・ます調で、非エンジニアにも分かる説明か。
- 一人称が自然で、`いかがでしたか`などの定型句がないか。
- 未確認の経験や成果を断定していないか。
- コードに秘密情報や個人情報を埋めていないか。
- 投資記事で銘柄推奨や利益保証をしていないか。
- スクール系でメリットだけを書かず、注意点も説明しているか。
- 無料部分だけでも読者が記事の価値を判断できるか。
- 有料部分に再現可能な具体性があるか。

最後に文字数の概算、前川が確認・追記すべき箇所、保存先を表示してください。
