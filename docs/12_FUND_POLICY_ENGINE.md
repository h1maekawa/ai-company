# Fund Department 投資判断エンジン 初期仕様書

更新日: 2026-07-19  
対象: 個人用AI会社 `personal-fund` / `/fund`  
位置づけ: `docs/11_INVESTMENT_DEPT_PLACEMENT.md` の次工程

## 1. 結論

Fund Manager AIは、利益を保証したり将来価格を断定するシステムではない。

目的は次の5点である。

1. 大きな損失と資金不足を避ける。
2. 購入・保有・売却を同じ基準で比較する。
3. 短期・中期・長期を混同しない。
4. 判断理由、反対材料、データ時点を必ず残す。
5. AIは候補を提示し、注文は本人が最終判断する。

## 2. 確定済みの前提

| 項目 | ルール |
|---|---|
| 短期 | 数日〜2か月以内 |
| 中期 | 2か月超〜6か月以内 |
| 長期 | 6か月超 |
| 目標配分 | 投資信託50%、個別株50% |
| 配分調整 | 一括売買せず、新規資金を優先して段階的に近づける |
| 1銘柄注意ライン | 個別株内30% |
| 1銘柄原則上限 | 個別株内40% |
| 注文 | 自動実行しない |
| 投資可能額 | Flow+または`capacity.md`から取得 |
| 会社用現金 | 今後の投資可能額計算では考慮しない |
| 月20万円 | 将来目標。毎月達成を強制する売買ノルマにしない |

## 3. 最優先原則

判定優先順位は次のとおりとする。

```text
資金安全性
→ データ完全性
→ 配分・集中リスク
→ 投資仮説
→ 価格・出来高・RVOL
→ 期待収益
```

上位条件を通過しない場合、下位のスコアが高くても購入候補にしない。

初期バージョンでは以下を禁止する。

- 信用取引、レバレッジ、空売り、オプション
- 本人承認なしの注文
- 月間利益目標を埋めるための取引
- RVOLだけを理由にした購入
- 値下がりだけを理由にしたナンピン
- データ不足時の強い断定
- 既存保有比率が高いという理由だけの全売却

## 4. 投資可能額

### 4.1 計算式

```text
投資可能額
= 現在の銀行残高
+ 次回判定日までの確定入金
- 未引落しカード利用額
- 固定費・予定支出
- 個人生活用バッファ
- 当月すでに投資した金額
```

```text
個人生活用バッファ
= 直近90日間の変動生活費を30日分に換算 × 1.2
```

家賃、確定カード引落し、予定支出は別項目で差し引き、生活用バッファへ重複計上しない。

### 4.2 購入停止条件

以下のいずれかで新規購入を停止する。

- 投資可能額が0円以下
- Flow+または`capacity.md`の更新が24時間超
- 残高、カード利用額、予定支出のいずれかが不明
- `confidence`が`low`
- `missing_data`に必須項目がある

## 5. 配分と集中管理

### 5.1 投資信託・個別株

| 個別株比率 | 判定 |
|---:|---|
| 45〜55% | 許容帯。銘柄条件を優先 |
| 40〜45% | 新規資金は個別株を優先候補 |
| 55〜60% | 新規資金は投資信託を優先候補 |
| 40%未満または60%超 | 強い配分警告。新規資金で調整 |

配分だけを理由に既存資産を売らない。税金、売買コスト、投資仮説、集中度を同時に確認する。

### 5.2 1銘柄上限

分母は個別株評価額とする。

| 比率 | 判定 |
|---:|---|
| 30%未満 | 通常 |
| 30〜40% | 集中注意。追加購入を厳格審査 |
| 40〜50% | 原則として追加購入停止 |
| 50%超 | 高集中警告 |

株価上昇で上限を超えた場合、即時売却しない。新規購入停止、新規資金による分散、段階売却の順に検討する。

### 5.3 テーマ上限

半導体、AI、クラウドセキュリティ等、同じ材料で同時に動きやすい銘柄をテーマとして集計する。

| 個別株内テーマ比率 | 判定 |
|---:|---|
| 60%未満 | 許容 |
| 60〜70% | 集中注意。テーマ内の追加購入を抑制 |
| 70%超 | テーマ内の新規購入を原則停止 |

現在のNVDA・MU合計は個別株内87.69%のため、初期状態では半導体の新規購入を原則停止する。既存株を機械的に売らず、他テーマへの新規投資、投資信託、利益確定条件の成立によって段階的に改善する。

### 5.4 ルックスルー集中度

投資信託・ETFの上位組入銘柄を展開し、個別株との重複を合算する。初期はデータ未取得なら`unknown`とし、表面上の集中度と実質集中度を混同しない。

## 6. 市場データ定義

### 6.1 RVOL

```text
RVOL20 = 当日出来高 ÷ 過去20営業日の平均出来高
```

当日は過去平均へ含めない。株式分割等の異常値調整後データを使用する。

取引時間中は、現在時刻までの累積出来高を過去20営業日の同時刻までの平均累積出来高で割る。日中途中の出来高を1日平均と直接比較しない。

| RVOL | 表示 |
|---:|---|
| 0.8未満 | 低調 |
| 0.8〜1.2 | 通常 |
| 1.2〜1.5 | 増加 |
| 1.5〜2.0 | 高い |
| 2.0以上 | 異常な注目 |

RVOLには必ず、価格方向、ニュース・決算、売買代金、スプレッドを併記する。

### 6.2 流動性

```text
ADTV20 = 過去20営業日の平均終値 × 平均出来高
spreadPct = (ask - bid) ÷ midpoint × 100
```

| 期間 | 最低ADTV20 | 最大スプレッド |
|---|---:|---:|
| 短期 | 2,000万ドル | 0.30% |
| 中期 | 1,000万ドル | 0.50% |
| 長期 | 500万ドル | 0.75% |

スプレッド不明の場合、短期は購入不可、中長期は警告付きとする。

### 6.3 データ鮮度

| データ | 新規購入に使える条件 |
|---|---|
| 株価・出来高 | 最新営業日またはリアルタイム |
| Flow+ | 24時間以内 |
| 決算 | 最新四半期を反映 |
| 会社予想・ガイダンス | 最新発表を反映 |
| ニュース | 発表日時と情報源を保持 |

必須データが古い場合は`WAIT_DATA`を返す。

## 7. 市場環境判定

市場環境はSPY等の広範指数で判定し、個別銘柄スコアとは分離する。

| 状態 | 条件の初期値 | 新規購入額係数 |
|---|---|---:|
| RISK_ON | 終値が200日線上、50日線が上向き | 1.00 |
| NEUTRAL | 上記の一方だけ成立 | 0.75 |
| RISK_OFF | 終値が200日線下、50日線が下向き | 0.50 |

市場環境だけで長期保有株を売却しない。新規購入量と短期判断に反映する。

## 8. 期間別スコア

各期間を100点満点で別々に評価する。同じ銘柄でも「長期BUY、短期WAIT」を許容する。

| 点数 | 判定 |
|---:|---|
| 75以上 | 購入候補。ゲートと上限を通過した場合のみ |
| 60〜74 | 監視 |
| 40〜59 | 保有なら要確認。新規購入しない |
| 39以下 | 回避・撤退候補 |

### 8.1 短期スコア

| 評価項目 | 点数 |
|---|---:|
| RVOL・出来高・売買代金 | 20 |
| 価格トレンド・支持抵抗 | 20 |
| 決算・ニュース等の材料 | 20 |
| 流動性・スプレッド | 15 |
| リスクリワード | 15 |
| 市場環境 | 10 |

- 決算2営業日前から当日までは、決算狙い戦略を別途有効化しない限り新規購入しない。
- RVOLが3.0以上かつ当日上昇率が10%以上なら`OVERHEATED`とし、追いかけ買いしない。
- エントリー価格から撤退基準までの損失額がリスク予算内であること。

### 8.2 中期スコア

| 評価項目 | 点数 |
|---|---:|
| 売上・利益・キャッシュフロー | 25 |
| 決算と会社予想の変化 | 20 |
| 業界成長・市場シェア | 15 |
| バリュエーション | 15 |
| 価格トレンド・需給 | 10 |
| 財務・外部リスク | 15 |

### 8.3 長期スコア

| 評価項目 | 点数 |
|---|---:|
| 市場の長期成長性 | 20 |
| 競争優位・参入障壁 | 20 |
| 売上・利益・FCF成長 | 20 |
| 財務健全性 | 15 |
| 経営・資本配分 | 10 |
| バリュエーション | 10 |
| 仮説崩壊条件の明確さ | 5 |

## 9. 購入数量

### 9.1 1判断の損失予算

| 期間 | 総投資資産に対する最大想定損失 |
|---|---:|
| 短期 | 0.50% |
| 中期 | 0.75% |
| 長期 | 1.00% |

```text
購入可能株数
= floor(損失予算額 ÷ 1株あたり撤退時損失額)
```

```text
最終購入上限
= min(
  Flow+投資可能額,
  リスク予算から算出した金額,
  1銘柄上限までの余地,
  テーマ上限までの余地
) × 市場環境係数
```

1株でもリスク予算または集中上限を超える場合、購入候補を0株とする。端株対応は証券会社の対応確認後に追加する。

### 9.2 段階購入

| 回 | 比率 | 条件 |
|---|---:|---|
| 1回目 | 40% | 初期条件通過 |
| 2回目 | 30% | 仮説維持、過熱なし |
| 3回目 | 30% | 決算・材料等で確認、または健全な押し目 |

値下がりだけでは2回目・3回目を実行しない。

### 9.3 注文価格

AIは成行注文を推奨せず、購入上限価格を提示する。指値は価格を制御できる一方、約定を保証しないことを画面に表示する。

## 10. ナンピン条件

以下をすべて満たす場合だけ追加購入候補にする。

- 値下がり以外の新しい根拠がある
- 対象期間のスコアが75点以上
- 投資仮説が維持されている
- 最新決算・ガイダンスが悪化していない
- 1銘柄上限・テーマ上限を超えない
- Flow+投資可能額がプラス
- 追加後の平均取得価格と撤退基準を再計算済み

## 11. 利益確定

利益率だけで全期間を同じように売らない。

### 11.1 短期

`R = エントリー価格 - 撤退価格`とする。

| 到達 | 提案 |
|---|---|
| +1R | 3分の1を利益確定候補 |
| +2R | さらに3分の1を利益確定候補 |
| 残り | 10日安値または2ATR相当を参考に追随 |

自動のストップ注文は出さず、価格アラートと本人確認を基本とする。

### 11.2 中期

次のいずれかで25〜50%の段階売却を検討する。

- 当初の目標材料が実現した
- バリュエーションが過去・同業比較で極端になった
- 2四半期先までの期待を先に織り込んだと判断した
- 1銘柄またはテーマ上限を超えた
- 中期スコアが60点未満になった

### 11.3 長期

含み益が大きいだけでは売却しない。次の場合に段階売却する。

- 長期スコアが60点未満
- 成長率・利益率・FCFの構造的悪化
- 競争優位または市場シェアの明確な低下
- 経営・資本配分への重大な懸念
- 1銘柄・テーマ上限の大幅超過
- 必要資金があり、他候補より売却影響が小さい

## 12. 損切り・撤退

### 12.1 短期

購入前に価格による撤退基準を必須入力する。

```text
撤退幅 = 1.5 × ATR14
下限5%、上限10%
```

終値で撤退基準を割った場合、翌営業日の売却候補とする。材料の重大悪化や不正等は即時再判定する。

### 12.2 中期

- 中期スコア60点未満
- 決算または会社予想の重大な下方修正
- 2四半期連続で主要KPI悪化
- 当初の材料が否定された
- 取得価格から15%超下落で強制再審査

再審査後もスコア60点未満なら全売却または50%縮小候補とする。

### 12.3 長期

取得価格から20%超の下落は自動売却ではなく強制再審査とする。

次のいずれかが明確なら撤退候補にする。

- 投資仮説の中核が崩壊
- 競争優位の持続性が失われた
- 財務悪化により成長投資の継続が困難
- 経営の信頼性に重大な問題
- 長期スコア50点未満

## 13. 急な現金需要

会社用現金の常時確保は行わない。ただし個人の急な現金需要が発生した場合に、当日、3営業日以内、1週間以内、1か月以内の期限別売却候補を出す。

AIは、現金化までの日数、流動性、投資スコア、投資仮説、集中度改善、含み損益、税金・コスト、売却後配分を点数化する。

原則順位:

1. 未投資の余剰現金
2. 仮説崩壊またはスコアの低い銘柄
3. 短期ポジション
4. 上限超過銘柄の一部
5. 中期ポジション
6. 長期主力株・投資信託

銘柄名だけで固定順位を作らず、その時点の税引後手取りと投資仮説で再計算する。

## 14. AIの出力

```yaml
ticker: NVDA
horizon: long
decision: HOLD
score: 82
confidence: medium
max_buy_jpy: 0
max_shares: 0
entry_zone: null
invalidation: "長期仮説の崩壊条件"
next_review_at: "次回決算後"
data_as_of: "2026-07-19"
reasons:
  - "根拠1"
counterarguments:
  - "反対材料1"
blocked_by:
  - "semiconductor_theme_limit"
missing_data: []
```

`decision`は以下に限定する。

- `BUY_CANDIDATE`
- `WAIT`
- `HOLD`
- `ADD_CANDIDATE`
- `TRIM_CANDIDATE`
- `EXIT_CANDIDATE`
- `WAIT_DATA`

「絶対上がる」「必ず勝てる」等の断定は禁止する。

## 15. 判断ログ

すべての提案について、提案日時、使用データと更新日時、期間、判断とスコア、購入上限、エントリー帯、仮説、反対材料、撤退条件、本人判断、約定、1週間・1か月・3か月・6か月後の結果を保存する。

結果だけで過去判断を評価せず、当時の情報とルールに照らして評価する。

## 16. 短期提案の検証期間

短期売買の実績がCSV上で確認できていないため、以下のどちらかを満たすまで`paper_mode=true`とする。

- 30日経過
- 20件の提案が決着

検証指標は勝率、平均利益/平均損失、Profit Factor、最大ドローダウン、期待値、ルール逸脱率とする。

実資金移行条件の初期値:

- Profit Factor 1.2以上
- 最大ドローダウン10%以下
- ルール逸脱率5%以下
- 最低20件

条件未達なら閾値を調整し、実資金移行しない。

## 19. セキュリティ

- `/fund`と全`/api/fund/*`でサーバー側認証を必須にする
- CSV本文、保有数量、家計情報をログへ出さない
- 市場データの情報源と取得日時を保存する
- プロンプトだけで金額・株数を決めず、決定論的なTypeScript関数で上限を計算する
- AI出力をスキーマ検証し、不正な値は`WAIT_DATA`へ落とす
- 金額は整数円、価格・為替はDecimal相当で扱う

## 20. テスト受入条件

- 投資可能額0円以下で購入株数が必ず0
- Flow+更新24時間超で`WAIT_DATA`
- 1銘柄40%超で追加購入額0円
- テーマ70%超で同テーマ追加購入額0円
- 上限超過だけでは自動売却しない
- RVOL単独では`BUY_CANDIDATE`にならない
- スプレッド不明の短期銘柄は購入不可
- 古い株価で購入候補を出さない
- 購入額は資金、損失予算、銘柄上限、テーマ上限の最小値を超えない
- 1株が上限を超える場合は0株
- 長期株は20%下落だけで自動売却しない
- 未認証の画面/APIアクセスを拒否
- CSV・保有資産・家計明細がログに出ない
- AIが注文APIを呼べない

## 22. 参考にした原則

- FINRAは、同じ銘柄・業種・相関資産・投信内の重複による集中リスクと、定期的な確認、投信の中身確認、流動性確認を挙げている。  
  https://www.finra.org/investors/insights/concentration-risk
- Investor.govは、資産配分が投資期間とリスク許容度に依存し、資産内・業種間の分散と投信内の組入確認が重要としている。  
  https://www.investor.gov/introduction-investing/getting-started/asset-allocation
- Investor.govは、指値注文は価格を制御できるが約定を保証せず、ストップ注文は発動後に成行となって想定価格からずれる可能性があるとしている。  
  https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/investor-bulletins-14  
  https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/investor-bulletins-15
