## 目的

- 特定銘柄の購入推奨ではなく、判断方法と学習過程を中心に記録する。

## 現在の投資方針・関心

- 投資信託と米国個別株を運用中。
- 半導体関連を重視している。
- 現在の主要な検討対象はNVIDIA（NVDA）、Micron Technology（MU）、CrowdStrike（CRWD）。
- 短期で売買できる活発さと、中長期で成長を期待できる事業性の両方を持つ銘柄を探したい。
- 同じティッカーでも、短期・中期・長期の投資理由を分けて管理する。
- 「絶対に伸びる」と断定せず、伸びる根拠・崩れる条件・想定期間を記録する。

## 前川さん専用の投資方針（2026-07-19ヒアリング反映）

### 背景

- 株式会社Crestixの役員として勤務している。
- 現在の給与だけでは、急な資金需要へ十分に対応しづらいという課題がある。
- 個別株は基本的に中長期の成長を目的として購入している。
- 一方で、急に資金が必要になった場合は、含み益のある個別株を短期・中期で売却して現金化する可能性がある。

### 現在の基本思想

- NVIDIA、Micron等の半導体関連は、今後の長期成長を期待して購入している。
- 「必ず勝つ」ではなく、長期保有できる根拠のある企業を選び、損失確率を下げたい。
- 長期保有を基本としながら、十分な含み益がある場合は、必要資金に応じて一部または全部を売却できる状態を作りたい。
- 短期・中期・長期は銘柄を完全に分けるのではなく、保有目的・資金需要・相場状況によって切り替わる可能性がある。

### システム上の重要な考え方

- 評価益は確定した現金ではないため、現在価格、為替、税金、手数料を反映した売却可能額を別表示する。
- 「長期保有銘柄だから売らない」と固定せず、急な必要資金と売却による機会損失を比較する。
- 「含み益があるから売る」とも固定せず、投資仮説、割高・割安、集中度、今後の材料を確認する。
- 会社資金需要は将来の購入制限には使わず、過去に個人の長期資産を会社都合で早期売却した履歴として残す。
- 売却時には利益額だけでなく、「資金需要がなければ保有を続けたか」「売却後に後悔が残ったか」を記録する。

## 前川さん専用AI会社へ追加する役割

### Flow+連携・投資可能額管理AI

目的: 家計簿アプリの収支から、その月に無理なく投資へ回せる金額を判断する。

管理項目:

- 現在の現金残高
- 毎月の最低必要生活費
- 近い将来に必要となる予定資金
- 当月の収入・入金予定
- 確定支出と未引落しカード利用額
- 固定費・一時的な予定支出
- 投資信託・個別株の売却可能額
- 税引前損益と概算税引後受取額
- 円貨・外貨の内訳
- 売却後に残るポートフォリオ
- 売却しない場合の代替資金調達案
- 売却後の納得度・後悔度

毎月の投資可能額は、固定額を本人に入力させず、次の考え方で計算する。

```text
当月の投資可能額
= 現在使える個人資金
+ 次回判定日までの入金予定
- 未引落しのカード利用額
- 近い将来の確定支出
- 個人生活用として残す金額

0円以下の場合
→その月は新規投資を見送る案を表示
```

CSVでは直近の投信積立は月15,000円だが、これを毎月固定の余剰資金とは扱わない。

1. 各銘柄を最初に買った理由は何か。
2. 現在もその購入理由は維持されているか。
3. 追加購入した理由と時期は何か。
4. 急に必要になる可能性がある金額は、およそ10万円、30万円、50万円、100万円以上のどれに近いか。

## 学習済みの用語

| 用語 | 簡単な意味 | 何を見るための数字か | 注意点 |
|---|---|---|---|
| 出来高（Volume） | 一定期間に売買が成立した株数 | 銘柄の売買が活発か | 多いだけでは上昇・下落方向は分からない |
| 売買代金（Dollar Volume） | 株価×出来高 | 実際にどれくらいの金額が動いたか | 株価が違う銘柄同士は出来高だけで比較しない |
| 平均出来高 | 過去一定期間の平均出来高 | その銘柄の通常時の活発さ | 期間によって数値が変わる |
| RVOL（Relative Volume／相対出来高） | 現在の出来高÷過去平均出来高 | 普段よりどれくらい注目・売買が増えたか | 高RVOLは買いサインではなく、異変・注目の検知 |
| 流動性（Liquidity） | 価格を大きく動かさず売買できる度合い | 売りたいときに売れるか、買いやすいか | 出来高だけでなく売買代金、スプレッド、板の厚さも関係する |
| ボラティリティ（Volatility） | 株価変動の大きさ | 短期間でどれくらい上下するか | 高いほど利益機会も損失リスクも大きい |
| ティッカー（Ticker） | 株式市場で使う銘柄コード | 銘柄を識別する | NVIDIA=NVDA、Micron=MU、CrowdStrike=CRWD |
| スプレッド | 最も高い買い注文と最も安い売り注文の差 | 売買コスト・流動性 | 広い銘柄は注文直後に含み損になりやすい |
| カタリスト | 株価が動くきっかけ | 決算、製品発表、政策等 | 好材料でも期待が織り込み済みなら上がらない場合がある |
| 投資仮説 | なぜその株が伸びると考えるか | 購入・保有理由 | 仮説が崩れた条件も同時に決める |
| コア・サテライト運用 | 投信等の分散資産を土台に、個別株等で上乗せを狙う方法 | ポートフォリオの役割分担 | サテライトが大きくなりすぎると全体の値動きを支配する |
| 集中リスク | 一つの銘柄・業種・テーマへ資産が偏るリスク | 同じ悪材料で複数資産が同時に下がり得るか | 商品数が多くても中身が重複すれば分散とは限らない |
| ルックスルー（中身ベース） | 投信やETFの組入銘柄まで展開して保有比率を見る方法 | 個別株との重複や実質集中度 | 最新の組入データと更新日が必要 |
| 含み益 | まだ売却していない資産の値上がり分 | 現在の利益余力 | 売却するまで確定せず、価格・為替で変動する |
| 実現益 | 売却によって確定した利益 | 実際に確定した運用成果 | 入出金や税・手数料と区別する |
| 段階売却 | 保有分を一度に全部売らず、一部ずつ売る方法 | 利益確定と上昇余地を両立させる | 残す数量と再購入条件を先に決める |
| 生活防衛資金 | 投資資産を売らずに生活・緊急支出へ充てる現金 | 下落時の強制売却を避けられるか | 売却可能な株式と即時利用できる現金は別物 |
| 機会損失 | 一つの選択により、別の選択から得られた可能性を失うこと | 売却・保有・乗換えの比較 | 後知恵でなく判断時点の情報で評価する |
| 資金需要売却 | 投資判断ではなく、必要な現金を作るために資産を売ること | 本来の投資仮説と別の理由で売ったか | 利益が出ていても、長期の上昇機会を失う場合がある |
| 指値注文 | 売買したい価格を指定して出す注文 | 目標価格で利確・購入できるか | 注文を出したことと約定したことは別。取引履歴で確認する |
| 約定 | 売買注文が成立した状態 | 実際に株数・現金が動いたか | 注文履歴ではなく約定履歴・残高・受渡金額で確認する |

## RVOLの理解

基本式:

```text
RVOL = 現在の出来高 ÷ 過去一定期間の平均出来高
```

例:

```text
過去20日平均出来高 = 100万株
今日の出来高       = 200万株
RVOL               = 2.0
```

意味: 普段の2倍の株数が売買されている。

AIでの使い方:

- いつもより注目されている銘柄を発見する。
- 決算・ニュース後に本当に取引が増えたか確認する。
- 株価上昇と出来高増加が同時に起きているか確認する。
- 暴落・投げ売りによる出来高増加も区別する。
- 高RVOLを単独で「買い」と判定しない。

## 似ているが異なる3つの観点

| 観点 | 問い | 主な指標 |
|---|---|---|
| 市場活動度 | どれくらい活発に取引されているか | 出来高、売買代金、RVOL、約定件数 |
| 流動性 | 希望に近い価格で売買しやすいか | 平均売買代金、スプレッド、板の厚さ |
| ボラティリティ | 株価がどれくらい大きく動くか | 値幅、ATR、標準偏差、前日比 |

重要な整理:

- 出来高が多い銘柄でも、値動きが小さい場合がある。
- 値動きが大きくても、出来高が少なければ流動性の低い危険な銘柄かもしれない。
- 活発な銘柄であることと、良い投資先であることは別。

## 投資AI会社へ反映する確定要件

### 市場活動度分析

- 出来高
- 売買代金
- 5日・20日・50日平均出来高
- RVOL（最優先で実装）
- 売買代金の増減
- 約定件数（取得可能な場合）
- 株式回転率
- 流動性
- 値動き

### 判定の順番

1. 現在活発に売買されているか。
2. 普段より活発になっているか。
3. 上昇・下落のどちらへ動いているか。
4. 活発化の原因は決算・ニュース・一時的な話題のどれか。
5. 売買しやすい流動性があるか。
6. 企業自体を保有する価値があるか。
7. 現在の価格で買うリスクが許容範囲か。

## 短期・中期・長期を分ける要件

同じ銘柄でも、投資期間ごとに別のスコアと投資仮説を持たせる。

### 短期

確定期間: 数日〜2か月以内

確認項目:

- RVOL
- 出来高・売買代金の急増
- 株価トレンド
- 決算・ニュース等のカタリスト
- 流動性
- ボラティリティ
- 購入価格と撤退条件

### 中期

確定期間: 2か月超〜6か月以内

確認項目:

- 売上・利益の成長
- 業績予想の上方・下方修正
- 業界の成長
- 複数四半期の決算推移
- 市場シェア
- 株価評価
- 金利・景気・半導体サイクル等の外部要因

### 長期

確定期間: 6か月超

確認項目:

- 市場そのものが成長するか
- 競争優位性が維持できるか
- 売上・利益・キャッシュフローが長期成長するか
- 財務体質
- 経営陣・研究開発
- 顧客・製品への依存
- 長期仮説が崩れる条件

### 全期間対応候補の判定

「短期でも長期でも使える」と一括断定せず、次を別々に表示する。

```text
短期適合度: 0〜100
中期適合度: 0〜100
長期適合度: 0〜100
総合コメント: それぞれの根拠と注意点
```

同じティッカーを短期枠と長期枠で持つ場合も、購入理由・投資額・売却条件を仮想ロット単位で分離する。

## 現在の調査候補

### NVIDIA（NVDA）

- AI・GPU・データセンター成長の長期仮説を検証する。
- 短期は決算、RVOL、売買代金、価格の過熱を確認する。
- Micronと同じ「AI設備投資・半導体」リスクへの集中を確認する。

### Micron Technology（MU）

- HBM・DRAM・NAND需要の成長を検証する。
- メモリ価格、供給量、在庫、平均販売価格の循環を確認する。
- SEC提出資料でも、供給増加に需要が追いつかない場合は平均販売価格が下落し得る点がリスクとして示されている。

### CrowdStrike（CRWD）

- サイバーセキュリティ市場とクラウド型プラットフォームの成長を検証する。
- NVIDIA・Micronとは別業種だが、成長株・高評価銘柄としての共通リスクを確認する。
- 短期の市場活動度と、長期の顧客維持・売上・キャッシュフローを分けて評価する。

## 要件決定状況（2026-07-19更新）

### 確定

1. 短期: 数日〜2か月以内
2. 中期: 2か月超〜6か月以内
3. 長期: 6か月超
4. 投資信託は既存の積立・保有運用を継続する。
5. 個別株は短期・中期・長期のいずれか一つに限定せず、3期間すべてを対象にする。

### 保有資産を分類してから一緒に決める

1. 短期・中期・長期それぞれの購入条件
2. 短期・中期・長期それぞれの売却・撤退条件
3. 現在の保有銘柄をどの期間・どの投資仮説で保有しているか
4. 同じティッカーを複数期間で保有する場合の仮想ロット分け
5. 短期・中期・長期の資金配分

### CSV分析で追加確定

1. 初期設計は中長期を主軸とし、短期売買は別枠・少額検証から始める。
2. 個別株の売却案は、全売却だけでなく段階売却を標準対応にする。
3. 証券口座の現金と銀行等の生活防衛資金を分けて管理する。
4. 個別株だけでなく、投信の組入銘柄まで展開した実質集中度を計算する。
5. 月20万円目標は、配当・売却益・元本取り崩しを分けて管理する。
6. 売買ごとに保有日数を計算し、本人定義の短期・中期・長期へ自動分類する。
7. 売却理由は、会社資金・個人生活資金・利確・仮説崩壊・資産配分変更から複数選択できるようにする。
8. 注文中・失効・取消・約定済みを区別し、本人の記憶と約定履歴が異なる場合はCSVを根拠に通知する。
9. 売却後に「もう少し持っておきたかった」等の納得度・後悔度を記録し、次回の売却ルール改善へ使う。
10. 会社用待機資金10万〜30万円の確保ルールは、今後の実費負担がなくなる見込みのため削除する。
11. 投資資産の目標配分は投資信託50%、個別株50%とする。
12. 毎月の投資可能額は固定せず、Flow+の収入・支出・残高・カード未引落し額から自動算出する。
13. Flow+のデータが古い、不完全な場合は具体的な投資金額を確定しない。
14. 投資可能額がある場合は個別株側へ寄せて50:50へ近づけるが、銘柄別上限・集中度・購入条件を必ず確認する。

## 初期運用ルールとして確定した内容（2026-07-19）

- 「負けない」は利益保証ではなく、大きな損失を避け、同じ基準で再現可能な判断を行うことと定義する。
- 個人生活用バッファは、直近90日間の変動生活費を30日分に換算した金額の1.2倍とする。
- 1銘柄は個別株内30%で注意、40%で追加購入を原則停止する。
- 半導体・AI等の1テーマは個別株内60%で注意、70%で同テーマの追加購入を原則停止する。
- RVOL20は当日出来高を過去20営業日の平均出来高で割る。当日は平均に含めない。
- RVOL2.0以上は異常な注目として扱うが、単独では買いサインにしない。
- 短期・中期・長期を別々に100点満点で評価し、75点以上を購入候補とする。
- 投資可能額、集中上限、データ鮮度、流動性を通過しなければ、75点以上でも購入候補にしない。
- 購入候補額は原則40%・30%・30%の3回に分ける。
- 値下がりだけを理由とするナンピンは禁止する。
- 短期は価格撤退基準と損失予算を購入前に決める。中長期は価格下落だけでなく、決算・投資仮説・競争優位の変化を確認する。
- 短期提案は30日または20件の決着までpaper modeとし、Profit Factor、最大ドローダウン、期待値、ルール逸脱率を測る。
- 詳細な実装仕様は`12_FUND_POLICY_ENGINE.md`にまとめる。
