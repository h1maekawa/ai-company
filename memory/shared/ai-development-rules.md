# AI開発 共通ルール

## 基本方針

- 既存機能を壊さない
- 大きく作り替える前に、まず小さく修正する
- 非エンジニアが理解できるように説明する
- 修正後は必ず確認方法を提示する
- 不明点がある場合は、仮説を置いて前に進める

## Claude Code / Codex への指示形式

開発AIに渡す要件は、以下の形式にする。

1. 目的
2. 現状の問題
3. 修正要件
4. 対象ファイル
5. 非対象ファイル
6. 完了条件
7. テスト方法
8. 注意点

## 安全ルール

- 本番DBを勝手に更新しない
- 顧客情報を外部送信しない
- `.env` やAPIキーを公開しない
- `NEXT_PUBLIC_` がついた環境変数に秘密情報を入れない
- 削除処理を行う場合は、先にバックアップまたはアーカイブを作る
- Obsidian Vaultやmemoryを削除する場合は、参照されていないことを確認する
